Projet Lend Me

Lend Me est une plateforme de crowdfunding.

Ce projet m’a permis de pratiquer des technologies vues au cours de cette année : Java et Angular. J’ai pu réaliser un projet dans sa globalité, me rendre compte et apprendre des divers points importants d’un tel chantier. Je comprends mieux pourquoi, pour des projets complexes, tel que des logiciels de clients, il est préférable de découper les tâches à réaliser entre divers membres d’une équipe. Il est en effet très difficile de tout maîtriser de la conception à la production d’une application. Cependant, j’ai apprécié l’opportunité de réaliser un tel challenge, malgré tous les défis à surmonter. Ce cheminement m’a permis d’améliorer ma vision d’un projet, des divers métiers impliqués dans la réalisation.


Pour le développement de mon application, je me suis inspirée des technologies utilisées lors de mon travail en entreprise, dans la cellule MCO (Maintienance en Condition Opérationnelle) :
j’ai donc fait le choix d’un front-end en langage TypeScript avec un framework Angular CLI 9,
d’un back-end en langage Java (jdk8) avec le framework Spring Boot,
ainsi qu’une base de données relationnelle MySql.
Ceci me permet de me conforter dans ces technologies, afin de me rapprocher de mon but professionnel : devenir développeuse full-stack.
Un framework est « un cadre de travail ». C’est un outil qui permet aux développeurs de de s’organiser et d’être plus efficace. L’avantage d’un framework est d’uniformiser l’environnement de travail, pour que chaque membre d’un projet puisse avoir la même base pour développer. Il est aussi nécessaire dans un projet donné de s’entendre sur les conventions de codage afin d’avoir un programme facilement compréhensible par tout nouveau membre intégrant l’équipe : lint, clean code, bonnes pratiques et patterns. J’ai une approche décentralisée car j’ai séparé les applications : front – back – BDD en vue de les conteneuriser dans un container Docker. Et une l’intégration continue (CI) qui assure le bon la bonne compilation du projet avant son déploiement en utilisant un pipeline Jenkins par exemple et le déploiement continu (CD).
Architecture d’ensemble de l'application CRUD Spring Boot & Angular 9 : Approche méthodologique utilisé dans l’architecture :
J’ai mis en place une architecture microservices avec un seul back-end qui expose les données via REST.
1/ REST (Representation State Transfer) est un type d’architecture qui définit un ensemble de contraintes à utiliser pour créer des services Web.
2/ J'ai utilisé des designs patterns comme MVC et Singleton.

3/ J'ai aussi travaillé sur la partie réseau :
La communication entre l’utilisateur et l’instance AWS,
La communication entre les conteneurs au sein de l’instance.
Diagramme réseau permettant de montrer où l’application est déployée du point de vue réseau :
4 containers :
Port : 3306 pour la BDD MySql
Port : 4200 pour le front-end Angular
Port : 8080 pour le back-end Java
Port : 9090 pour Jenkins

  • Langages: Java & Angular

Projet Vidéothèque

Vidéothèque est un projet créé par moi-même en langage Symfony5, avec une base de donnée Mysql.

Ma base de donnée Mysql, me permet la persistence des données entrées par l'administrateur de la plateforme Vidéothèque et la consultation des films par les utilisateurs.
Pour accéder au contenu de la plateforme, un vistiteur peut venir s'inscrire sur la page principale de Vidéothèque et consulter les fiches des films mises en ligne par les administrateurs.
Un administrateur qui se connecte en tant : adminTest@gmail.com et avec le mot de passe : helloWord, peut accéder à son interface d'adminstration. Cliquer ici pour visulaiser la ligne de base de données pour tester le dasboard d'administration de la plateforme Vidéothèque Son dashboard AdminTesteur présente 2 onglets : l'onglet utilisateurs avec les informations relatives aux utilisateurs et l'onglet Média avec les films et les différentes actions possibles (fonctions CRUD : Create - Delete - Update - Read):
1- créer ses fiches de vidéothèques en cliquant sur "Ajouter un film",
2- les lire en cliquant sur "Consulter la fiche",
3- les modifier en cliquant sur "Editer la fiche"
4- et/ou les supprimeren cliquant sur "Supprimer la fiche".
De plus, vous pouvez voir le code source de mon projet de Vidéothèque en Symfony 5 sur mon Github : https://github.com/magalierambla/Videotheque-symfony
